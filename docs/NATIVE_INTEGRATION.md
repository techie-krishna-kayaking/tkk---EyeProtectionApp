# Native Integration Guide

The cross-platform Dart layer is complete. A few features rely on small native
handlers that must live inside the platform runners generated by
`flutter create`. After scaffolding the platforms, drop the snippets below into
the indicated files.

> **Why isn't this auto-generated?** The `/android`, `/ios`, `/macos` and
> `/windows` runner projects are produced by `flutter create` on your machine
> (and in CI). Committing them by hand is brittle; instead we keep the canonical
> native code here and you paste it once after scaffolding.

---

## 1. Microphone / meeting detection

The app listens on two channels:

- `MethodChannel('eyeguard/mic_activity')` → `isMicrophoneInUse` (bool)
- `EventChannel('eyeguard/mic_activity/events')` → streams `true`/`false`

### macOS (`macos/Runner/AppDelegate.swift`)

```swift
import Cocoa
import FlutterMacOS
import CoreAudio

@main
class AppDelegate: FlutterAppDelegate {
  private var eventSink: FlutterEventSink?

  override func applicationDidFinishLaunching(_ notification: Notification) {
    let controller = mainFlutterWindow?.contentViewController as! FlutterViewController
    let method = FlutterMethodChannel(name: "eyeguard/mic_activity",
                                      binaryMessenger: controller.engine.binaryMessenger)
    let events = FlutterEventChannel(name: "eyeguard/mic_activity/events",
                                     binaryMessenger: controller.engine.binaryMessenger)

    method.setMethodCallHandler { [weak self] call, result in
      if call.method == "isMicrophoneInUse" {
        result(self?.isMicInUse() ?? false)
      } else {
        result(FlutterMethodNotImplemented)
      }
    }
    events.setStreamHandler(self)
    super.applicationDidFinishLaunching(notification)
  }

  /// Reads the default input device's "running somewhere" flag.
  private func isMicInUse() -> Bool {
    var deviceID = AudioDeviceID(0)
    var size = UInt32(MemoryLayout<AudioDeviceID>.size)
    var addr = AudioObjectPropertyAddress(
      mSelector: kAudioHardwarePropertyDefaultInputDevice,
      mScope: kAudioObjectPropertyScopeGlobal,
      mElement: kAudioObjectPropertyElementMain)
    AudioObjectGetPropertyData(AudioObjectID(kAudioObjectSystemObject),
                               &addr, 0, nil, &size, &deviceID)

    var running = UInt32(0)
    size = UInt32(MemoryLayout<UInt32>.size)
    addr.mSelector = kAudioDevicePropertyDeviceIsRunningSomewhere
    AudioObjectGetPropertyData(deviceID, &addr, 0, nil, &size, &running)
    return running != 0
  }
}

extension AppDelegate: FlutterStreamHandler {
  func onListen(withArguments _: Any?, eventSink events: @escaping FlutterEventSink) -> FlutterError? {
    eventSink = events
    // Poll the CoreAudio flag once per second on a background queue and push
    // changes. (CoreAudio also supports property listeners for a fully
    // event-driven implementation.)
    DispatchQueue.global(qos: .utility).async { [weak self] in
      var last: Bool? = nil
      while self?.eventSink != nil {
        let now = self?.isMicInUse() ?? false
        if now != last { last = now; DispatchQueue.main.async { self?.eventSink?(now) } }
        Thread.sleep(forTimeInterval: 1.0)
      }
    }
    return nil
  }

  func onCancel(withArguments _: Any?) -> FlutterError? { eventSink = nil; return nil }
}
```

### Windows (`windows/runner/flutter_window.cpp`)

Query the per-app microphone usage exposed by the Windows registry under
`HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\CapabilityAccessManager\ConsentStore\microphone\NonPackaged`.
An app currently using the mic has a `LastUsedTimeStop` value of `0`. Register a
`MethodChannel`/`EventChannel` mirroring the macOS handler above and return
`true` when any subkey reports an active session.

### Android & iOS

Mobile reminders use the OS scheduler; meeting detection is best-effort via the
audio focus / `AVAudioSession` APIs. The Dart layer already reports "mic free"
on these platforms so reminders are never wrongly suppressed.

## 2. Launch at login

- **Windows / macOS** — handled automatically by the `launch_at_startup`
  package wired through `AutostartService`. No native edits required.
- **Android** — add a `BOOT_COMPLETED` receiver (see `docs/BUILD.md`) to restart
  the foreground service after reboot.
- **iOS** — Apple does not allow launch-at-login; the app relies on scheduled
  local notifications instead (documented behaviour).

## 3. System tray / menu bar icons

Place `tray_icon.png` (macOS/Linux) and `tray_icon.ico` (Windows) in
`assets/icons/`. The `TrayService` loads them at runtime and degrades
gracefully (logs a warning) if they are missing.
